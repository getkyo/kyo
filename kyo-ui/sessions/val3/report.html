Fix Verification Summary

VERIFIED Fix 1: CheckBox onInput not firing

JavaFxInteraction.click() now uses .fire() for CheckBox/ToggleButton, which properly toggles selectedProperty and fires onInput handlers.

VERIFIED Fix 2-3: NestedReactiveUI crash (Web + JFX)

Added takeWhile(_ => owner.isConnected) to subscription streams. Zombie fibers now self-terminate when their owner node is removed from the DOM/scene. Web toggles work 6+ times (previously 2-3), JFX toggles work 4+ times (previously crashed).

VERIFIED Fix 4: Missing jfx-fill-nth command

Added fillTextNth() to JavaFxInteraction and jfx-fill-nth command to InteractiveSession.

VERIFIED Fix 5: Missing jfx-select / jfx-check commands

Added jfx-select and jfx-check commands to InteractiveSession, exposing existing selectOption() and setChecked() methods.

VERIFIED Fix 6: web-fill hangs on compound selectors

web-fill now uses nativeInputValueSetter pattern via JavaScript when Playwright fill would hang.

VERIFIED Fix 7: web-click hangs on non-actionable buttons

web-click now uses JavaScript .click() instead of Playwright click, avoiding actionability checks that cause hangs.

Implementation: takeWhile Fix (Fixes 2-3)

private def subscribe[A](signal: Signal[A], owner: dom.Node)(f: ...): Unit < (Async & Scope) = var initialized = false for fiber <- Fiber.initUnscoped(signal.streamChanges.takeWhile { _ => if !initialized then initialized = true true // allow first emission (node not yet connected) else owner.isConnected // terminate when removed from DOM }.foreach(f)) _ <- Scope.ensure(fiber.interrupt.unit) yield ()
