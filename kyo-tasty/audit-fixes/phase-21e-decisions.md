# Phase 21e Decisions

## Empty-ZLIB byte construction

The plan specified a 6-byte empty-payload ZLIB stream. Investigation showed that
`PortableInflate.inflate` requires at least 6 bytes to pass the length guard, but a
6-byte stream (2 header + 4 Adler-32 trailer) leaves 0 bytes for DEFLATE blocks. The
while loop in `inflate` would read bits from the Adler-32 bytes, which is incorrect.

The shortest valid ZLIB stream encoding an empty payload is 8 bytes:

    0x78 0x9C  -- CMF/FLG: CM=8 (deflate), CINFO=7, checksum (0x789C % 31 == 0)
    0x03 0x00  -- DEFLATE fixed-Huffman block: BFINAL=1, BTYPE=01 (fixed), then
                  EOB symbol 256 (7 bits = 0000000 in LSB-first order)
    0x00 0x00 0x00 0x01  -- Adler-32 big-endian for empty input = 1

Generated by `java.util.zip.DeflaterOutputStream` compressing an empty byte array
(verified with JVM code). The PortableInflate pure-Scala path and the JVM
InflaterInputStream path both accept this stream and produce an empty array.

## Depth-guard approach: visited-set cycle detection

Two options were considered:

1. **Depth counter bounded by `entries.length`**: Simple integer counter. Stops after
   `entries.length` recursive steps. Easy to implement but does not detect the exact
   cycle site; it falls through after exceeding the bound.

2. **Visited-set cycle detection**: A `Set[Int]` of entry indices already on the
   current resolution path. If the next entry index is already in the set, a cycle is
   detected immediately at the first back-edge.

Option 2 was chosen because:
- It detects cycles precisely at the back-edge, not after an arbitrary depth.
- It is more correct for the expected use case (EXTref chains are short; a pickle entry
  count of N does not mean there are N-deep chains).
- No default parameters are needed; `visited: Set[Int]` is passed explicitly as a new
  parameter at both call sites, starting with `Set.empty[Int]`.

The signature change is:
  Before: `def resolveExtFqn(entryIdx: Int, nameTable: ..., entries: ...): String`
  After:  `def resolveExtFqn(entryIdx: Int, nameTable: ..., entries: ..., visited: Set[Int]): String`

Both call sites in `decodeExtRef` and `decodeExtModClassRef` pass `Set.empty[Int]`.
Recursive calls pass `visited + entryIdx`.

## Cycle test details (Test 10)

Fixture: two-entry pickle.
- Entry 0: TERMname "Foo"
- Entry 1: EXTref(nameRef=0, ownerRef=1)  <- ownerRef is the entry's own index

Resolution trace:
1. `decodeExtRef` for entry 1: ownerRef=1, calls `resolveExtFqn(1, ..., Set.empty)`.
2. `resolveExtFqn(1, ..., Set.empty)`: not in visited, not a name entry, entry 1 is
   EXTref. Reads ownerRef=1 again. `nextVisited = Set(1)`.
3. `resolveExtFqn(1, ..., Set(1))`: `visited.contains(1)` is true; returns "".
4. `ownerFqn` is filtered-empty (Absent). `leafName` = "Foo" (nameRef=0). Returns "Foo".
5. Back in step 2: `ownerFqn = Present("Foo")`. Returns "Foo.Foo".
6. `decodeExtRef`: `fqn = "Foo" + "." + "Foo" = "Foo.Foo"`.

The test asserts that the resulting symbol has FQN "Foo.Foo" (finite, not an infinite
string), proving the cycle guard terminates the recursion.
