# Research finding Q-005

Question: Does the M4 snapshot format extension warrant a minor (add-only) or major (invalidating) version bump?
Type: research-knowable
Budget: 10 minutes (used: 4)

## Answer

A minor version bump (add-only) is sufficient. The existing layout has no field widths that need to widen. Section index entries already store offset and length as 8-byte Int64LE fields (`SnapshotWriter.scala:206-208`), so no index-level widening is needed. Within section payloads, all count and length fields are 4-byte Int32LE (`SnapshotWriter.scala:227,261,285`), and Int32 is more than adequate for parent/typeParam/declaration counts per symbol. The PARENTS and MEMBERS sections already exist as registered names in `SnapshotFormat.sectionNames` (line 67) and are written as empty byte arrays today (`SnapshotWriter.scala:141-142`); populating them is an add-only operation. A new TYPEPARAMS section, if needed, is also add-only because the reader skips unknown sections by name via the sectionMap lookup. No existing field width, no existing section ordering, and no existing record layout changes. The versioning policy at `SnapshotFormat.scala:42-44` specifies that minor bumps cover add-only sections; old snapshots load with new sections empty. This matches exactly: a reader at minorVersion 2 will find PARENTS/MEMBERS/TYPEPARAMS absent and fill _parents/_typeParams/_declarations with Chunk.empty, identical to the existing fallback at `SnapshotReader.scala:174-177`.
